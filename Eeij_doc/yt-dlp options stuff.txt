mpv Server

Since you're a Python user on Linux, you can leverage the JSON-IPC interface.
This allows you to treat a running mpv instance like a remote server that you can control via a Unix socket.

1. Launch mpv with the IPC Server

First, you must tell mpv to create a "listener" (a socket file) so your Python script has a place to send commands.

Command Line:
Bash

mpv --input-ipc-server=/tmp/mpv-socket --idle "https://youtu.be/CQx6m_g6R1k"

(The --idle flag keeps mpv open even if the video ends, which is useful for testing scripts).

2. The Python Manager Script

You don't need fancy libraries; Python’s standard socket and json modules can handle this perfectly.
Python

import socket
import json
import os

class MPVController:
    def __init__(self, socket_path='/tmp/mpv-socket'):
        self.socket_path = socket_path

	def send_command(self, *args):
		"""Formats and sends a JSON command to the mpv socket."""
		if not os.path.exists(self.socket_path):
			print(f"Error: Socket {self.socket_path} not found. Is mpv running?")
			return None

		# Build the JSON payload
		payload = {"command": list(args)}

		# Connect to the Unix Domain Socket
		with socket.socket(socket.AF_UNIX, socket.SOCK_STREAM) as client:
			client.connect(self.socket_path)
			client.sendall((json.dumps(payload) + '\n').encode('utf-8'))

			# Receive response (mpv sends a JSON response for every command)
			response = client.recv(4096)
			return json.loads(response.decode('utf-8'))

# Example Usage:
mpv = MPVController()

# 1. Toggle Pause
mpv.send_command("cycle", "pause")

# 2. Show a message on the video screen
mpv.send_command("show-text", "Hello from Python!", "2000")

# 3. Adjust Volume
mpv.send_command("add", "volume", "10")

# 4. Load a new video (with your cookies from your config)
mpv.send_command("loadfile", "https://youtu.be/dQw4w9WgXcQ")

3. Why this is powerful for Python users:

    Real-time Monitoring: You can write a Python loop to query properties like ${time-pos} or ${metadata} to build your own custom HUD or logging system.

    Automation: You can trigger Python actions when a video ends by watching the eof-reached property.

    Multiple Instances: By changing the path (e.g., /tmp/mpv-socket-1, /tmp/mpv-socket-2), you can control several independent players from a single script.

Troubleshooting

    The Trailing Newline: mpv requires each JSON command to end with a newline (\n). If you forget it, the player will sit there waiting forever.

    Permissions: Since /tmp/ is world-writable, it's a safe bet, but ensure your script has read/write permissions to whatever path you choose.
